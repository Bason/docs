How To Solve ZOO Issues

If you are having trouble or issues with the ZOO, this can have numerous causes. The ZOO has a build in system check, to detect some of the problems. But there are some commen issues that might arise:

Faulty interface actions - Javascript Errors

If you are having trouble with functions in the ZOO, this might be due to a jQuery conflict. Please read {doc: javascript-errors text: this tutorial} on how to detect and resolve the problem.

Unstyled content - no CSS being loaded

This might be due to file permission issues. Please read {doc: file-permission-issues-3 text: this tutorial} on how to detect and resolve the problem.

Blank Page

This is most likely due to a php error being thrown. Please enable the display_errors parameter in your php.ini. Only with an error message the problem can be found.

Another way to check for PHP errors, is to have a look at your PHP error log. The PHP log shows information on errors and warnings thrown by PHP. Your host should have configured PHP to write that information to a log rather than output them on screen.

Please check this guide to PHP error messages for webdesigners for a more thourough view on the subject.

Repair Corrupt Database Table

Occasionally database tables become corrupt and you are no longer able to access the data. Usually, you can fix the table so you can access the data again. Doing this in phpMyAdmin requires only a few steps.

  • Login to phpMyAdmin.
  • Choose the affected database.
  • In the control panel, you should see a list of your database tables. Check the box by the table that needs repair.
  • At the bottom of the window just below the list of tables, there is a drop down menu. Choose Repair Table.
