Javascript errors

How to detect Javascript errors in general?

One way to detect Javascript errors on your page is to use the browser Firefox and an addon called Firebug. Firebug will give you quite a few details and hints at what is going wrong. This will really help with debugging the error. For more detailed instructions on how to use firebug see this guide over at sixrevisions.com.

Javascript conflicts

You need to make sure jQuery is only loaded once on your site. Otherwise you will get JavaScript conflicts and all effects may stop working like Accordion Menus, Google Maps, etc..

Warp itself always loads the latest jQuery version. So what if you 3rd party extension also loads jQuery?

How to prevent multiple jQuery instances?

There is an ongoing discussion in the Joomla community on how to prevent those conflicts from happening. We have already taken steps and measures by implementing a widely accepted solution. We register through JApplication whether jQuery is loaded or not. In case the 3rd party extension loads the jQuery library you can use the following code snippet to prevent our ZOO extension, Widgetkit or Warp template from loading it twice:

// load jQuery, if not loaded before
if (!JFactory::getApplication()->get('jquery')) {
	JFactory::getApplication()->set('jquery', true);
	// add jQuery
	...
}

So get in contact with the 3rd party extension provider and kindly ask them to implement this.
