Config.xml

The config.xml defines the Warp7 theme settings.

In the theme settings, you can control basic preferences for the theme layouts, widgets and menus. Each setting is represented by a menu tab and its options and is defined in the config.xml.

<fields> elements

<fields> elements define menu tabs, representing each setting.

<?xml version="1.0" encoding="utf-8"?>
<config>
    <fields name="Settings" icon="uk-icon-cogs">
        ...
    </fields>
    <fields name="Layouts" icon="uk-icon-columns">
        ...
    </fields>
    <fields name="Modules" icon="uk-icon-th">
        ...
    </fields>
    <fields name="Menus" icon="uk-icon-reorder">
        ...
    </fields>
    <fields name="Information" icon="uk-icon-info-sign">
        ...
    </fields>
</config>

Add a new menu item

A <fields> element will add a new tab to the theme settings. The value of the name attribute will render the tab name.

<fields name="MY-MENUITEM" icon="uk-icon-info-sign">
    ...
</fields>

<field> elements

<field> elements inside <fields> elements define the options for the menu tabs.

Basic XML types

You can use basic XML types to create new options for your theme settings. This could be for example a radio button, a checkbox or an input field.

Type Description
section Defines the section for a description text.
select Defines a dropdown list.
radio Defines a radio button.
checkbox Defines a checkbox.
textarea Defines a multiline text input.

Special XML types

There are also some Warp7 specific types available: layouts, verify, styles, compile and info.

Markup example

// Section field
<field type="section" name="Development" description="Always uncheck following options for your production website." />

// Checkbox field
<field type="checkbox" name="dynamic_style" value="1" label="Allow styles to be loaded by URL." />

// Select field
<field type="select" name="compression" default="0">
    <option value="0">None</option>
    <option value="1">Minify</option>
    <option value="2">Minify + Data URIs</option>
    <option value="3">Minify + Data URIs + Gzip</option>
</field>

// Textarea field
<field type="textarea" name="tracking_code" class="uk-form-width-large" rows="8" />

// Radiobutton field
<field type="radio" name="my_option" value="1" label="Description" />

Add new options

To create a new option, add a field element to the setting, i.e. inside the fields element, to which you want to add your option. You can also copy an existing one and modify it to your needs.

<field type="checkbox" name="MY-OPTION" value="1" label="Description" />

Add functionality to an option

To give your new option a functionality, you need to respond to the name attribute of the <field> element with the $this['config']->get('MY-OPTION') syntax.

<?php if($this['config']->get('MY-OPTION')): ?>
<p>Hello World</p>
<?php endif; ?>;

This is the corresponding field in the config.xml.

<field type="checkbox" name="MY-OPTION" value="0" label="Description." />

config.json

You can find all your saved settings in the minified config.json file. With every new saving process, the file will be overridden.

Warp themes provide a separate config.default.json file, where the default theme settings are stored. This gives you the opportunity to restore your backend to the default settings and backup your own config.json file, making your settings update proof.

NOTE For a better overview, use the JSON Formatter - this will bring your minified file into easily readable markup.
