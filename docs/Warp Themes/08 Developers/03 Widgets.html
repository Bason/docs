Widgets/Modules

Learn how to create new widget positions, layouts and styles.

Adding a new position for modules or widgets to your theme is fairly straightforward. You have to name a position and define where the position gets rendered within the {doc: theme-layout text: theme layout}.

Add a new position

Add a new position to the templateDetails.xml for Joomla or the theme.xml for WordPress to make your content management system know it exists. The new option will then be displayed in your theme settings.

<position>top-a</position>
<position>MY-POSITION</position>
<position>bottom-a</position>

Add settings

With the settings attribute you can control the options for the widget appearance of your position. You will find these in the Modules or Widgets panel of your theme settings. Available values are class, style, icon, badge and display. Additionally, title and assignment are available in WordPress.

<position settings="">MY-POSITION</position>

NOTE If no settings attribute is given, your position automatically shows all available options. If you use the settings attribute without adding a value, it won't be displayed in the Modules or Widgets panel at all.

Get more information about settings in the {doc: widgets text: Widgets/Modules} chapter of the Get Started section.

Theme settings for positions

To make the theme options of the Settings and Layouts panels for your position available, you have to add your new position to the config.xml.

Set default widget rendering

By adding a new <row> element to the Settings section the option to choose a default style for your position in the Settings panel of your administration will be displayed.

<fields name="Settings">
    ...
    <field type="table" name="panel_default">
        <rows label="Position">
            <row>MY-POSITION</row>
            ...

Set default layout rendering

By adding a new <row> element to the Layouts section the options to set a default layout, dividers, the preserve function or width and alignment of your position will be displayed in the Layouts panel.

<fields name="Layouts">
    ...
    <field type="table" name="grid">
        <rows label="Position">
            <row>MY-POSITION</row>
            ...

Take a look at {doc: config text: this chapter} for more information on the config.xml. To get an overview of all widget/module options in the theme settings take a look at the {doc: settings text: Settings} and {doc: layouts text: Layouts} docs.

Render position

The layouts/theme.php provides the basic markup of the theme. Here you can define where widget positions get rendered and add new ones.

<?php if ($this['widgets']->count('MY-POSITION')):?>
    <section class="tm-MY-POSITION uk-grid" data-uk-grid-match="{target:'> div > .uk-panel'}">
        <?php echo $this['widgets']->render('MY-POSITION', array('layout'=>$this['config']->get(
        'grid.MY-POSITION.layout'))); ?>
    </section>
<?php endif;?>

NOTE You only need to add the $this['config']->get('grid.MY-POSITION.layout') syntax, if you have declared an option in the config.xml.

Take a look at {doc: theme-layout text: Theme Layout} for a better understanding of the widget render methods.
