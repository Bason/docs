System overrides

This is an introduction to system files and how to override the output.

To override the system output, you need to add the system specific directory to your theme, copy the relevant files and modify the parts that you want.

Modify or add system files

Check, if your files already exist in the layouts/ directory. If it is empty or the required files are missing, copy them from the warp/systems/joomla/layouts/ or warp/systems/wordpress/layouts/ directory to the layouts/ directory and edit them to your needs. If you want to learn more about the system specific output files, visit the Joomla and WordPress Documentation.

Alternatively, you can override the files in the custom/ directory in order to make your changes update proof. Make sure only to copy the files you want to change. The other files will be loaded automatically from the system defaults.

Custom template files for WordPress

You can use different layouts for template files, for example pages, by creating files in your theme folder with the name prefix page- followed by the ID or page-slug.

  • page-{id}.php
  • page-{slug}.php

Let's say you want custom content for a page with the ID 80. Create a file in your custom/ folder and name it page-80.php with the following content.

<?php get_header(); ?>

Put your content here...

<?php get_footer(); ?>

In the backend you can even assign a custom style to the page, so you'll get a completely different look.
