Customize the Notification Pages

The notification pages available in Warp themes are white-label, minimalist designs that fit perfectly with any of our themes and serve as an override to the system defaults.

There are three types of notification pages available:

Error Page

When a visitor requests a missing page, the error page layout will be displayed with a 404 error title and description text.

How to Customize the Error Page (Joomla)

The HTML output of the error page is located in /warp/layouts/error.php within your theme. If you would like to edit the language output, simply edit /warp/systems/joomla/language/YOUR_LANGUAGE_CODE/YOUR_LANGUAGE_CODE.tpl_warp.ini subbing out YOUR_LANGUAGE_CODE with your proper language code.

How to Customize the Error Page (WordPress)

The HTML output of the error page is located in /warp/layouts/error.php within your theme. If you would like to edit the language output, visit the /languages directory of your theme. Copy the file en_US.po and rename it to YOUR_LANGUAGE_CODE.po. Now you can translate the .po file to your language using Poedit and compile it to YOUR_LANGUAGE_CODE.mo subbing out YOUR_LANGUAGE_CODE with your proper language code. For further instructions please take a look at the Translating WordPress tutorial.

Offline Mode Page

In the event you need to perform maintenance on your website, activating the offline mode page will notify your users that your site is currently unavailable. This is extremely useful when upgrading or testing out new features on your website.

How to Customize the Offline Mode Page (Joomla)

The HTML output of the offline mode page is located in /warp/systems/joomla/layouts/offline.php within your theme. If you would like to edit the language output, simply edit /warp/systems/joomla/language/YOUR_LANGUAGE_CODE/YOUR_LANGUAGE_CODE.tpl_warp.ini subbing out YOUR_LANGUAGE_CODE with your proper language code.

Browser Upgrade Page

It's the new-age people! Modern web technologies like HTML5 and CSS3 are used all around the web to progressively enhance websites, cutting down on the dependencies of JavaScript, Flash, etc. Unfortunately, not all browsers are able to take advantage of these tactics, the worst offender being IE6. Since Warp6 does not support IE6, the browser upgrade page is used to display a strong yet simple message, notifying the user they should upgrade their browser for a better viewing experience. This can be turned off via the theme settings page.

How to Customize the Browser Upgrade Page (Joomla)

The HTML output of the browser upgrade page is located in /warp/layouts/error.php within your theme. If you would like to edit the language output, simply edit /warp/systems/joomla/language/YOUR_LANGUAGE_CODE/YOUR_LANGUAGE_CODE.tpl_warp.ini subbing out YOUR_LANGUAGE_CODE with your proper language code.
