Override system files

This tutorial shows you how to override system files in Joomla and create custom pages in WordPress.

Override Joomla system files

Lets say you want to customize the layout of the content Joomla component. To do that, just copy the com_content directory from /warp/systems/joomla.1.x/layouts to the theme /layouts directory and change the files to your needs. Or if you are using your own style copy the directory to /styles/YOUR_STYLE/layouts. Further this way you won't lose any of your changes after an update of your theme. Make sure to only copy the files you want to change, all other layout files will be automatically loaded from the system defaults.

WordPress custom pages

You can use different layouts for pages by creating files in your theme folder with a "page-" as prefix in the name followed by the ID or page-slug.

  • page-{id}.php
  • page-{slug}.php

Lets say you want a custom layout for a page with ID 80. Create a file in your theme folder and name it page-80.php with the following content.

// get warp
$warp = Warp::getInstance();

// load main template file, located in /layouts/template-80.php
echo $warp['template']->render('template-80');

Then go to your themes /layouts directory and copy and rename template.php to template-80.php. Now you can start to customize template-80.php as you like to.
