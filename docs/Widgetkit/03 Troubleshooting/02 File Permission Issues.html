Permission Issues

On your webserver, different files and directories have permissions that specify who and what can read, write, modify and access them. This is important because Widgetkit need access to write to files in to the cache directory to enable its caching functionality needed for galleries etc.

So please make sure after installing widgetkit, please make sure that the cache folder that Widgetkit uses is writable.

Joomla : /cache/widgetkit
WordPress : /wp-content/plugins/widgetkit/cache/

To change the permission on your remote host, use your favorite FTP client.

Always avoid 777 permissions. If your webserver has problems with the CHMOD 755, you can also try 775 and lastely 777 in this order.

Solve File Permission Issues

So how do you change the permissions for the files and folders? Here are two step-by-step guides.

The first guide explains how to use your favorite FTP client such as the FileZilla FTP Client. When using a FTP client, sometimes the ownership of the files/folders you changed the permissions for, changes to the user currently logged in via ftp. This can cause problems. Therefore there is a second solution, which uses eXtplorer, a Joomla component.

Using your favorite FTP client such as FileZilla FTP Client.

  1. Download FileZilla FTP Client and install it.
  2. Use FileZilla FTP Client to login to your ftp account.
  3. Right click on the ZOO folder and click File Attributes.
  4. In the Change File Attributes dialog, check the options as shown below or type 755. Make sure the Recurse into subdirectories option is checked. Then hit the OK button.
Tip to avoid some trouble: After applying new permissions some FTP clients require the file view to be refreshed. Otherwise the old permissions will continue to show...

For Joomla: Using eXtplorer component

  1. Download eXtplorer and install it. eXtplorer is a file manager. It allows you to edit, delete, copy, rename, archive and unpack files/directories directly on your server.
  2. In the Joomla administration, go to the eXtplorer component and navigate to the ZOO folder. Check the ZOO folder.
  3. Right click on the folder and choose Change Permissions
  4. Check the options as shown for the ftp client above. Make sure the Recurse into subdirectories option is checked. Then hit the Save button.
